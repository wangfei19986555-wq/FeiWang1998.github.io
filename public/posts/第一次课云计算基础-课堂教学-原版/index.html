第一部分：项目陈述（10 min）#

🎬 课堂导入：一个你每天都在经历的场景#

想一想：你今天用手机了吗？

  • 早上刷抖音看视频 → 视频存谁那？→ 存云端
  • 用微信发照片给朋友 → 照片经过哪里？→ 云端服务器
  • 打开淘宝逛了一圈 → 淘宝页面跑在哪？→ 阿里云
  • 你用百度网盘存文件 → 文件实际存在哪里？→ 云端存储

🔑 核心认知：你不是在使用自己的手机/电脑的算力，而是在使用「云」上的计算资源。

❓ 那么，到底什么是云计算？#

一句话总结：通过网络随时随地获取你需要的计算资源，就像拧开水龙头就有水一样。

🏠 生活类比：从「打井」到「自来水」#

时代方式类比
传统 IT自己买服务器、建机房 → 像每家自己挖井取水💰 投入大、维护累
云计算按需从云端获取算力 → 像拧开水龙头用水💡 按量付费、省心省力
传统模式：买服务器 → 装系统 → 部署应用 → 维护 → 扩容难 → 闲置浪费
云模式：  开账号   → 选配置 → 部署应用 → 自动维护 → 弹性扩容 → 按量付费

📖 云计算的定义（正式版）#

云计算（Cloud Computing） 是一种基于网络的超级计算模式，将计算任务分布在大量计算机构成的资源池上，使各种应用系统能够根据需要获取计算力、存储空间和各种软件服务。

☁️ 为什么叫「云」？#

云的特征对应含义
云很大计算集群规模庞大
云可伸缩资源动态扩展/收缩
云的边界模糊用户无需知道具体物理位置
云飘忽不定底层硬件对用户透明
云确实存在数据中心真实存在于某处

🔬 狭义 vs 广义#

维度狭义云计算广义云计算
定义通过网络提供 IT 基础设施资源通过网络提供各种服务
比喻像自来水厂供水像整个公共服务网络
范围硬件 + 软件 + 平台包含一切可在线获取的服务
例子买阿里云服务器用钉钉、腾讯文档、网盘

🎓 讲给学生的总结：
云计算不是一种新技术，而是一种新概念、新模式。
它的核心思想是：以互联网为中心，让计算资源像水电一样流通。

🔗 三层关系#

IaaS 是底层基础 → PaaS 建立在 IaaS 之上 → SaaS 面向最终用户

    SaaS  ← 最终用户使用（如：你用网盘存文件）
     ↓
    PaaS  ← 开发者使用（如：调用云数据库 API）
     ↓
    IaaS  ← 运维人员管理（如：配置虚拟服务器）

2.2 四大部署模式#

模式说明优点缺点典型场景
公有云 🔵第三方提供，互联网访问成本低、免维护数据不在本地个人/创业公司
私有云 🟢专属单一组织使用安全可控自己出钱运维银行/政府
混合云 🟠公有 + 私有组合兼顾安全与弹性架构复杂大型企业
社区云 ⚪几个组织共享分摊成本较少见行业协会/联盟

💡 一句话选型指南：没钱没数据 → 公有云；有钱有数据 → 私有云；两者都要 → 混合云。

三、云计算的特征#

特征含义大白话
🔄 按需自助服务不用找人工，自己在线开通像网购一样自助下单
🌐 广泛的网络访问手机/电脑/平板都能连有网就能用
📦 资源池化多用户共享计算资源池像水库供水给全城
⚡ 快速弹性需要就扩，不用就缩双11扩容，平时缩回来
📊 按量计费用多少付多少钱像交水费，看表计价

🎯 核心价值：把 IT 从「固定资产」变成「流动费用」，企业不需要为「万一要用」而买一堆闲置服务器。

第三部分：必备知识 — 虚拟化技术（25 min）#

❓ 为什么云计算需要虚拟化？#

核心问题：一台物理服务器很强大，但 15% 的利用率太浪费。怎么办？
答案：虚拟化！一台物理机 → 多台虚拟机，利用率可达 60%+。

┌─────────────────────────────────────────────────┐
│              虚拟化前 vs 虚拟化后                     │
├────────────────────┬────────────────────────────┤
│     虚拟化前        │        虚拟化后               │
│ 1 台物理机          │  1 台物理机                   │
│ 1 个操作系统        │  N 个虚拟机（各自独立 OS）       │
│ 利用率 ~15%         │  利用率 ~60%+                  │
│ 应用绑定硬件        │  应用可自由迁移                  │
│ 资源争抢冲突        │  资源隔离、互不影响               │
└────────────────────┴────────────────────────────┘

一、什么是虚拟化？#

虚拟化：把物理资源转变为逻辑上可管理的资源，打破物理结构的壁垒。

🏢 类比：一栋大楼 → 隔成多个房间 → 每个房间独立出租，互不干扰。
物理服务器 = 大楼，虚拟机 = 房间，Hypervisor = 隔墙和门禁系统。

二、虚拟化体系结构#

┌─────────────────────────────────────────┐
│   虚拟机1     │   虚拟机2    │  虚拟机3    │  ← Guest（客户机）
│  (Windows)    │  (Linux)    │  (Linux)   │
├─────────────────────────────────────────┤
│           Hypervisor (VMM)              │  ← 虚拟机监控器
│     管理虚拟机运行、资源分配和隔离        │
├─────────────────────────────────────────┤
│          物理硬件（CPU/内存/硬盘/网卡）    │  ← Host（宿主机）
└─────────────────────────────────────────┘

Hypervisor 的两种类型#

类型原生型（裸机型）Native宿主型（托管型）Hosted
运行位置直接跑在硬件上跑在操作系统之上
性能损耗较小较大
典型产品VMware ESXi、Hyper-VVMware Workstation、VirtualBox
适用场景🏢 企业生产环境🏠 个人实验/测试
类比独栋别墅，地基自己打公寓楼，依赖大楼结构

🎓 记忆技巧：
原生型 = 裸奔 = Bare-metal → 企业用
宿主型 = 寄生 = Hosted → 学习用（我们的课程用 VMware Workstation）

三、虚拟化的分类#

虚拟化
├── 平台虚拟化 ───── 针对计算机和操作系统
│   ├── 服务器虚拟化    → VMware ESXi、Hyper-V、XenServer
│   └── 桌面虚拟化      → 远程桌面、云桌面
│
├── 资源虚拟化 ───── 针对特定计算资源
│   ├── 存储虚拟化      → 多块硬盘 → 逻辑存储池
│   └── 网络虚拟化      → 网格计算、SDN
│
└── 应用程序虚拟化 ── 针对软件运行环境
    ├── Java 虚拟机（JVM）
    └── Docker 容器（后续课程重点！）

四、全虚拟化 vs 半虚拟化#

维度全虚拟化半虚拟化
是否需要修改 OS❌ 不需要✅ 需要
用户感知不感知（以为在真机）能感知（知道在虚拟环境）
兼容性强，支持任意 OS弱，需定制 OS
性能略低（Hypervisor 开销）较好
未来趋势🏆 主流辅助角色

五、云计算中的主流虚拟化技术#

技术类型特点与 Docker 的关系
KVM全虚拟化Linux 内核原生、OpenStack 默认Docker 早期基于 LXC（类似理念）
Xen全/半虚拟化开源、Amazon AWS 早期使用—
Hyper-V全虚拟化微软出品、Windows 集成Docker Desktop 依赖 Hyper-V（Windows）
VMware ESXi全虚拟化企业级、业界标杆vSphere 集成 Docker
容器（Docker）🌟 操作系统级虚拟化轻量、秒级启动本课程核心！

🔥 重要提示：Docker 是操作系统级虚拟化（容器），不是传统虚拟机。
本质区别：虚拟机虚拟硬件 → 每个 VM 有自己的 OS；容器共享宿主机 OS → 只隔离应用进程。

六、KVM 虚拟化架构（深入理解）#

Linux 平台上最重要的虚拟化方案，也是理解云平台底层的关键。

KVM 两大组件#

组件运行位置职责
KVM 内核模块内核空间CPU/内存虚拟化管理
QEMU用户空间I/O 设备模拟、虚拟机配置

KVM 虚拟磁盘格式#

格式特点什么时候用
raw直读直写、最简单、支持增减容量大数据存储
qcow2按需分配空间、支持快照/压缩/加密默认选择 ✅
qedqcow2 改进版尚未成熟

💡 本课程实操用 VMware Workstation（宿主型、适合学习），
企业环境用 KVM/ESXi（原生型、高性能）。

🛠 1.3.1 VMware Workstation 安装#

软件准备：VMware Workstation Pro / Player
系统要求：Windows 10/11 64位，至少 8GB 内存，支持虚拟化技术

安装前检查#

3
4

# 检查 CPU 是否支持虚拟化
systeminfo | findstr "虚拟化"

⚠️ 常见问题：如果显示"已禁用"，需要进入 BIOS 开启 Intel VT-x 或 AMD-V。

安装步骤（教师演示）#

  1. 下载 VMware Workstation → 双击安装程序
  2. 接受许可协议 → 选择安装路径
  3. 勾选「增强型键盘驱动程序」
  4. 等待安装完成 → 输入许可证密钥
  5. 重启计算机

🖥 1.3.2 创建虚拟机#

典型配置方案（以 CentOS 7 为例）#

配置项推荐值说明
操作系统CentOS 7 / Ubuntu 20.04选 Linux
CPU2 核不低于 1 核
内存4 GB不低于 2 GB
硬盘40 GB（qcow2 按需分配）存储为单个文件
网络NAT 模式可访问外网
虚拟化引擎勾选 Intel VT-x/AMD-V嵌套虚拟化支持

关键步骤图解#

1. 新建虚拟机 → 典型（推荐）
2. 稍后安装操作系统
3. 选择 Linux → CentOS 7 64位
4. 命名虚拟机 → 选择存储位置
5. 指定磁盘容量 → 「将虚拟磁盘存储为单个文件」
6. 自定义硬件 → 挂载 ISO 镜像
7. 完成 → 开启虚拟机 → 按提示安装系统

🎯 课堂练习：每人安装 VMware Workstation 并创建一台 CentOS 虚拟机。
