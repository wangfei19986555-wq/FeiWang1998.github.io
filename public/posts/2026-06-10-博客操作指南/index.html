博客位置：C:\Users\wf980\Desktop\myblog\

🗂 文件结构#

myblog/
├── hugo.toml              ← 📌 博客设置（标题、链接等）
├── content/
│   ├── about.md           ← 📌 「关于我」页面
│   └── posts/
│       └── xxx.md         ← 📌 每篇文章就是一个 .md 文件
├── static/                ← 放图片、文件等
└── .github/workflows/     ← 自动部署配置（不用管）

✏️ 修改文章或页面#

操作步骤#

  1. 用 VS Code 打开 C:\Users\wf980\Desktop\myblog\ 整个文件夹
  2. 找到要改的文件：
    • content\posts\xxx.md → 文章
    • content\about.md → 关于页面
  3. 直接编辑 Markdown，和写教案一样
  4. 保存后，浏览器 http://localhost:1313 自动刷新看到效果（需要 Hugo 在运行）

---
title: "文章标题"            ← 改这里修改标题
date: 2026-06-02             ← 日期
draft: false                 ← false = 发布, true = 草稿（不显示）
tags: ["云计算", "入门"]      ← 标签，可以加多个
categories: ["云计算"]        ← 分类
ShowToc: true                ← 是否显示目录
---

📝 写一篇新文章#

在终端输入：

1
2
3
4

cd "C:\Users\wf980\Desktop\myblog"

# 创建新文章
hugo new posts/2026-06-10-docker-intro.md

文件名格式建议：日期-文章主题.md

创建后：

  1. 打开新生成的 .md 文件
  2. 把 draft: true 改成 draft: false
  3. 往下写正文

👀 本地预览#

1
2

cd "C:\Users\wf980\Desktop\myblog"
hugo server -D

浏览器打开 http://localhost:1313

参数作用
-D包含草稿文章
--port=1313指定端口（默认 1313）

⚠️ 保持终端窗口不关，每次保存文件页面自动刷新。

🔑 三句话总结#

  1. 所有内容都在 content/ 里，就是写 Markdown
  2. hugo server -D 保持不关，本地随时预览
  3. 满意了就 git push，自动发布
