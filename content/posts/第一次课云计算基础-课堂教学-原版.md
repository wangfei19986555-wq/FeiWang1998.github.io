## 🎬 课堂导入：一个你每天都在经历的场景

> **想一想**：你今天用手机了吗？

- 早上刷**抖音**看视频 → 视频存谁那？→ 存云端
- 用微信发**照片**给朋友 → 照片经过哪里？→ 云端服务器
- 打开**淘宝**逛了一圈 → 淘宝页面跑在哪？→ 阿里云
- 你用**百度网盘**存文件 → 文件实际存在哪里？→ 云端存储

🔑 **核心认知**：你不是在使用自己的手机/电脑的算力，而是在使用「云」上的计算资源。

---

## ❓ 那么，到底什么是云计算？

> **一句话总结**：通过网络随时随地获取你需要的计算资源，就像拧开水龙头就有水一样。

### 🏠 生活类比：从「打井」到「自来水」

| 时代 | 方式 | 类比 |
|------|------|------|
| **传统 IT** | 自己买服务器、建机房 → 像每家自己挖井取水 | 💰 投入大、维护累 |
| **云计算** | 按需从云端获取算力 → 像拧开水龙头用水 | 💡 按量付费、省心省力 |

```
传统模式：买服务器 → 装系统 → 部署应用 → 维护 → 扩容难 → 闲置浪费
云模式：  开账号   → 选配置 → 部署应用 → 自动维护 → 弹性扩容 → 按量付费
```

---

## 📖 云计算的定义（正式版）

**云计算（Cloud Computing）** 是一种基于网络的超级计算模式，将计算任务分布在大量计算机构成的**资源池**上，使各种应用系统能够根据需要获取计算力、存储空间和各种软件服务。

### ☁️ 为什么叫「云」？

| 云的特征 | 对应含义 |
|---------|---------|
| 云很大 | 计算集群规模庞大 |
| 云可伸缩 | 资源动态扩展/收缩 |
| 云的边界模糊 | 用户无需知道具体物理位置 |
| 云飘忽不定 | 底层硬件对用户透明 |
| 云确实存在 | 数据中心真实存在于某处 |

---

## 🔬 狭义 vs 广义

| 维度 | 狭义云计算 | 广义云计算 |
|------|-----------|-----------|
| **定义** | 通过网络提供 IT 基础设施资源 | 通过网络提供各种服务 |
| **比喻** | 像自来水厂供水 | 像整个公共服务网络 |
| **范围** | 硬件 + 软件 + 平台 | 包含一切可在线获取的服务 |
| **例子** | 买阿里云服务器 | 用钉钉、腾讯文档、网盘 |

> 🎓 **讲给学生的总结**：  
> 云计算不是一种新技术，而是一种**新概念、新模式**。  
> 它的核心思想是：**以互联网为中心，让计算资源像水电一样流通**。

## 二、云计算的基本概念

### 2.1 三大服务模式（重点！⭐）

> 这 3 个概念是整个课程的基石，请务必理解！

```
┌──────────────────────────────────────────────────────┐
│                    🍕 吃披萨的类比                       │
├──────────┬───────────────────┬─────────────────────────┤
│ 自己在家做 │     点外卖         │      去餐厅吃            │
│ 买面粉    │ 别人做好送上门       │  什么都不用管             │
│ 买烤箱    │ 你只需准备餐具       │  张嘴就吃                 │
│ 买配料    │ 吃完不用洗碗         │  吃完走人                 │
│ 自己烤    │                    │                          │
├──────────┼───────────────────┼─────────────────────────┤
│  IaaS    │      PaaS          │       SaaS               │
│ 基础设施  │     平台即服务       │     软件即服务             │
└──────────┴───────────────────┴─────────────────────────┘
```

| 服务模式 | 英文全称 | 你做什么 | 云厂商做什么 | 典型产品 |
|---------|----------|---------|-------------|---------|
| **IaaS** | Infrastructure as a Service | 自己装系统、部署应用 | 提供服务器/网络/存储 | 阿里云ECS、AWS EC2 |
| **PaaS** | Platform as a Service | 写代码即可 | 提供开发平台+中间件 | Google App Engine |
| **SaaS** | Software as a Service | 直接用 | 提供完整软件 | Gmail、钉钉、Office 365 |

#### 📊 三种模式对比

| 维度 | IaaS | PaaS | SaaS |
|------|------|------|------|
| **灵活性** | ⭐⭐⭐⭐⭐ 最高 | ⭐⭐⭐ 中等 | ⭐ 最低 |
| **开发量** | ⭐⭐⭐⭐⭐ 最多 | ⭐⭐ 较少 | 无需开发 |
| **控制力** | 完全可控 | 只控应用 | 不可控 |
| **适用场景** | 运维能力强的大企业 | 创业团队快速开发 | 普通用户/中小企业 |
| **类比** | 买车自己开 | 打车（司机开） | 坐公交车 |

#### 🔗 三层关系

> IaaS 是底层基础 → PaaS 建立在 IaaS 之上 → SaaS 面向最终用户

```
    SaaS  ← 最终用户使用（如：你用网盘存文件）
     ↓
    PaaS  ← 开发者使用（如：调用云数据库 API）
     ↓
    IaaS  ← 运维人员管理（如：配置虚拟服务器）
```

### 2.2 四大部署模式

| 模式 | 说明 | 优点 | 缺点 | 典型场景 |
|------|------|------|------|---------|
| **公有云** 🔵 | 第三方提供，互联网访问 | 成本低、免维护 | 数据不在本地 | 个人/创业公司 |
| **私有云** 🟢 | 专属单一组织使用 | 安全可控 | 自己出钱运维 | 银行/政府 |
| **混合云** 🟠 | 公有 + 私有组合 | 兼顾安全与弹性 | 架构复杂 | 大型企业 |
| **社区云** ⚪ | 几个组织共享 | 分摊成本 | 较少见 | 行业协会/联盟 |

> 💡 **一句话选型指南**：没钱没数据 → 公有云；有钱有数据 → 私有云；两者都要 → 混合云。

## 三、云计算的特征

| 特征 | 含义 | 大白话 |
|------|------|--------|
| 🔄 **按需自助服务** | 不用找人工，自己在线开通 | 像网购一样自助下单 |

# 第三部分：必备知识 — 虚拟化技术（25 min）

---

## ❓ 为什么云计算需要虚拟化？

> **核心问题**：一台物理服务器很强大，但 15% 的利用率太浪费。怎么办？  
> **答案**：虚拟化！一台物理机 → 多台虚拟机，利用率可达 60%+。

```
┌─────────────────────────────────────────────────┐
│              虚拟化前 vs 虚拟化后                     │
├────────────────────┬────────────────────────────┤
│     虚拟化前        │        虚拟化后               │
│ 1 台物理机          │  1 台物理机                   │
│ 1 个操作系统        │  N 个虚拟机（各自独立 OS）       │
│ 利用率 ~15%         │  利用率 ~60%+                  │
│ 应用绑定硬件        │  应用可自由迁移                  │
│ 资源争抢冲突        │  资源隔离、互不影响               │
└────────────────────┴────────────────────────────┘
```

---

## 一、什么是虚拟化？

**虚拟化**：把物理资源转变为逻辑上可管理的资源，打破物理结构的壁垒。

> 🏢 **类比**：一栋大楼 → 隔成多个房间 → 每个房间独立出租，互不干扰。  
> 物理服务器 = 大楼，虚拟机 = 房间，Hypervisor = 隔墙和门禁系统。

---

## 二、虚拟化体系结构

```
┌─────────────────────────────────────────┐
│   虚拟机1     │   虚拟机2    │  虚拟机3    │  ← Guest（客户机）
│  (Windows)    │  (Linux)    │  (Linux)   │
├─────────────────────────────────────────┤
│           Hypervisor (VMM)              │  ← 虚拟机监控器
│     管理虚拟机运行、资源分配和隔离        │
├─────────────────────────────────────────┤
│          物理硬件（CPU/内存/硬盘/网卡）    │  ← Host（宿主机）
└─────────────────────────────────────────┘
```

### Hypervisor 的两种类型

| 类型 | 原生型（裸机型）Native | 宿主型（托管型）Hosted |
|------|----------------------|----------------------|
| **运行位置** | 直接跑在硬件上 | 跑在操作系统之上 |
| **性能损耗** | 较小 | 较大 |
| **典型产品** | VMware ESXi、Hyper-V | VMware Workstation、VirtualBox |
| **适用场景** | 🏢 企业生产环境 | 🏠 个人实验/测试 |
| **类比** | 独栋别墅，地基自己打 | 公寓楼，依赖大楼结构 |

> 🎓 **记忆技巧**：  
> 原生型 = 裸奔 = Bare-metal → 企业用  
> 宿主型 = 寄生 = Hosted → 学习用（我们的课程用 VMware Workstation）

---

## 三、虚拟化的分类

```
虚拟化
├── 平台虚拟化 ───── 针对计算机和操作系统
│   ├── 服务器虚拟化    → VMware ESXi、Hyper-V、XenServer
│   └── 桌面虚拟化      → 远程桌面、云桌面
│
├── 资源虚拟化 ───── 针对特定计算资源
│   ├── 存储虚拟化      → 多块硬盘 → 逻辑存储池
│   └── 网络虚拟化      → 网格计算、SDN
│
└── 应用程序虚拟化 ── 针对软件运行环境
    ├── Java 虚拟机（JVM）
    └── Docker 容器（后续课程重点！）
```

---

## 四、全虚拟化 vs 半虚拟化

| 维度 | 全虚拟化 | 半虚拟化 |
|------|---------|---------|
| **是否需要修改 OS** | ❌ 不需要 | ✅ 需要 |
| **用户感知** | 不感知（以为在真机） | 能感知（知道在虚拟环境） |
| **兼容性** | 强，支持任意 OS | 弱，需定制 OS |
| **性能** | 略低（Hypervisor 开销） | 较好 |
| **未来趋势** | 🏆 主流 | 辅助角色 |

---

## 五、云计算中的主流虚拟化技术

| 技术 | 类型 | 特点 | 与 Docker 的关系 |
|------|------|------|-----------------|
| **KVM** | 全虚拟化 | Linux 内核原生、OpenStack 默认 | Docker 早期基于 LXC（类似理念） |
| **Xen** | 全/半虚拟化 | 开源、Amazon AWS 早期使用 | — |
| **Hyper-V** | 全虚拟化 | 微软出品、Windows 集成 | Docker Desktop 依赖 Hyper-V（Windows） |
| **VMware ESXi** | 全虚拟化 | 企业级、业界标杆 | vSphere 集成 Docker |
| **容器（Docker）** | 🌟 操作系统级虚拟化 | 轻量、秒级启动 | 本课程核心！ |

> 🔥 **重要提示**：Docker 是**操作系统级虚拟化（容器）**，不是传统虚拟机。  
> 本质区别：虚拟机虚拟硬件 → 每个 VM 有自己的 OS；容器共享宿主机 OS → 只隔离应用进程。

---

## 六、KVM 虚拟化架构（深入理解）

> Linux 平台上最重要的虚拟化方案，也是理解云平台底层的关键。

### KVM 两大组件

| 组件 | 运行位置 | 职责 |
|------|---------|------|
| **KVM 内核模块** | 内核空间 | CPU/内存虚拟化管理 |
| **QEMU** | 用户空间 | I/O 设备模拟、虚拟机配置 |

### KVM 虚拟磁盘格式

| 格式 | 特点 | 什么时候用 |
|------|------|-----------|
| **raw** | 直读直写、最简单、支持增减容量 | 大数据存储 |
| **qcow2** | 按需分配空间、支持快照/压缩/加密 | **默认选择** ✅ |
| **qed** | qcow2 改进版 | 尚未成熟 |

> 💡 **本课程实操用 VMware Workstation**（宿主型、适合学习），  
> **企业环境用 KVM/ESXi**（原生型、高性能）。

## 🛠 1.3.1 VMware Workstation 安装

> **软件准备**：VMware Workstation Pro / Player  
> **系统要求**：Windows 10/11 64位，至少 8GB 内存，支持虚拟化技术

### 安装前检查

```powershell
# 检查 CPU 是否支持虚拟化
systeminfo | findstr "虚拟化"

# 或通过任务管理器 → 性能 → CPU → 查看「虚拟化：已启用」
```

> ⚠️ **常见问题**：如果显示"已禁用"，需要进入 BIOS 开启 Intel VT-x 或 AMD-V。

### 安装步骤（教师演示）

1. 下载 VMware Workstation → 双击安装程序
2. 接受许可协议 → 选择安装路径
3. 勾选「增强型键盘驱动程序」
4. 等待安装完成 → 输入许可证密钥
5. 重启计算机

---

## 🖥 1.3.2 创建虚拟机

### 典型配置方案（以 CentOS 7 为例）

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 操作系统 | CentOS 7 / Ubuntu 20.04 | 选 Linux |
| CPU | 2 核 | 不低于 1 核 |
| 内存 | 4 GB | 不低于 2 GB |
| 硬盘 | 40 GB（qcow2 按需分配） | 存储为单个文件 |
| 网络 | NAT 模式 | 可访问外网 |
| 虚拟化引擎 | 勾选 Intel VT-x/AMD-V | 嵌套虚拟化支持 |

### 关键步骤图解

```
1. 新建虚拟机 → 典型（推荐）
2. 稍后安装操作系统
3. 选择 Linux → CentOS 7 64位
4. 命名虚拟机 → 选择存储位置
5. 指定磁盘容量 → 「将虚拟磁盘存储为单个文件」
6. 自定义硬件 → 挂载 ISO 镜像
7. 完成 → 开启虚拟机 → 按提示安装系统
```

> 🎯 **课堂练习**：每人安装 VMware Workstation 并创建一台 CentOS 虚拟机。
